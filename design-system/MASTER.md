# Kasa вЂ” Design System (Master)

> Source of truth for Kasa's UI. Generated by applying the `ui-ux-pro-max`
> ruleset to the Kasa PRD. Page-specific overrides live in `design-system/pages/`.

## Product framing

- **Type:** Education / language-learning **mobile app**, voice-first.
- **Audience:** low-literacy and nervous speakers (Akosua, Kwame) + literate
  visitors (Sarah). Design for zero literacy; literate users still win (PRD 6.2).
- **Tone:** warm, encouraging, local. "Patient older sibling," never a strict
  teacher (PRD 5.5).
- **Platform:** Android first, React Native. Respect Material touch/nav idioms.

## Style

- **Style:** warm, friendly, **flat with soft elevation** (not glass, not
  brutalist). Rounded, generous, tactile.
- **Motif:** Ghanaian вЂ” kente stripes for the streak, adinkra-inspired shapes,
  market/trotro scenes for packs. "Local, not generic" (PRD 6.5).
- **Effects:** soft shadows for cards/mic; radius 16вЂ“24; no heavy gradients that
  hurt readability (`trend-emphasis`, `elevation-consistent`).

**Rules:** semantic tokens only, never raw hex in components
(`color-semantic`). Colour is never the only signal вЂ” feedback pairs colour with
a word and per-word highlighting (`color-not-only`, PRD 5.2).

## Typography (`typography` in tokens)

- Scale: 16 / 20 / 22 / 26 / 34. Body **20px** (well above the 16px mobile
  minimum вЂ” audience skews older / low-literacy).
- Weights: headings 800, labels 700, body 500. Line-height ~1.4вЂ“1.5.
- One family for now (system); if adding a face, pick a friendly humanist sans
  (e.g. Nunito/`Baloo 2` for warmth) вЂ” pair later, don't block MVP.

## Layout & touch (CRITICAL)

- **Touch targets в‰Ґ 44pt** вЂ” `TAP_MIN = 64` for primary controls; use `hitSlop`
  where an icon is smaller (`touch-target-size`).
- 4/8pt spacing rhythm (`spacing` scale). Safe areas respected via
  `react-native-safe-area-context` (`safe-area-awareness`).
- One primary CTA per screen; secondary actions visually subordinate
  (`primary-action`).
- Mobile-first; test at 375px and in the largest Dynamic Type size.

## Motion

- Micro-interactions 150вЂ“300ms; press feedback within 100ms (`tap-feedback-speed`).
- Screen transitions: fade / directional slide, spatially consistent.
- Respect `prefers-reduced-motion` (`reduced-motion`).

## Accessibility (CRITICAL вЂ” this audience needs it most)

- Every button has an `accessibilityLabel`; voice-first means labels are also
  **spoken** on long-press (already implemented in `BigButton`, `PackTile`).
- Contrast в‰Ґ 4.5:1 for text (ink on cream passes).
- Screen-reader order matches visual order; feedback uses text + colour + audio.
